##### java.lang 클래스

자바 제공하는 것중에 기본이 되는 패키지 -> java.lang 패키지

###### 대표적인 클래스?
Object  : 자바 객체 부모
String   : 문자열
Integer, Long, Double : 래퍼 타입
Class : 클래스 메타 정보
System  : 시스템 기본 기능

import java.lang.System 생략 가능

##### Object 클래스

부모가 없으면 목시적으로 Object 클래스를 상속받음(= extends object)

=> 자바에서 모든 객체의 부모는 Object


Object는 필요한 공통 기능을 제공
- toString()
- equals()
- getClass()

다형성의 기본 구현
: Object는 모든 객체를 전부 다 담을 수 있음.

##### Object 다형성
Object는 모든 객체의 부모이므로
Dog dog = new Dog();
-> Object dog = new Object(); 가능
 
##### 한계
obj instanceof Dog dog -> Object에는 dog객체 메서드가 객체 다운캐스팅으로 불러와야 함.
=> Object로 객체를 호출하려면 다운 캐스팅 과정 필요
=> 메서드 오버라이딩 활용 불가능

##### Object 배열

*Object 인스턴스 생성 가능
size 메서드 : 배열에 담긴 객체 수를 셈.

Object 타입으로 배열을 만든다면, 모든 타입의 객체를 담을 수 있음
-> 클래스 추가/변경 시 메서드 수정 불필요

toString()
객체의 이름과 객체의 참조값 출력해줌

obj.toString -> 오버라이딩 된 메서드 존재하는 지 확인 한 후 해당 메서드 호출
                 -> 존재하지 않으면 기본 toString 호출

Integer.toHexString(System.identityHashCode(dog1))
=> 객체의 참조값 출력

##### Object와 OCP
A클래스가 구체적인 특정 클래스를 사용하는 것 -> A클래스는 특정 클래스에 의존한다고 표현

ObjectPrinter 클래스는 Object클래스에 의존
-> 구체적인 것이 아니라 추상적인 것에 의존

##### OCP원칙
open : 새로운 클래스를 추가하고, 오버라이딩으로 확장 가능
closed :  새로운 클래스를 추가해도 ObjectPrinter는 변경할 필요 없음

System.out.println 메서드도 Object 매개변수 사용하고, 내부에서 toStirng 호출함
=> 모든 객체 정보 편리하게 출력 가능

정적 의존관계 VS 동적 의존관계
코드만 보고 이해할 수 있는 의존 관계 -> 정적 의존관계
런타임에 확인할 수 있는 의존관계 -> 동족 의존관계

##### equals() 
동일성(identify) : 연산자 == 를 사용하여 두 객체의 참조가 동일한 객체가 가리키고 있는 지 확인
=> 물리적으로 같은 메모리에 있는 객체 인스턴지 인지 확인
동등성(Equality) : equals()메서드를 사용하여 두 객체가 논리적으로 동등한지 확인
=> 논리적으로 같은 지 확인

동등성 개념은 각각 클래스마다 다름
=> 동등성 비교를 위해서는 equals() 메서드를 재정의 해야함

* 문자열 비교는 == 이 아니라 equals() 사용해야함
* 정확한 equals메소드는 인텔리제이로 생성 가능.

equals 메서드 규칙
- 반사성
- 대칭성
- 추이성
- 일관성
- null 과 비교

* 동일성 비교가 항상 필요한 것은 아님 

##### 불변 객체
자바 데이터 타입
기본형(primitive Type) : 하나의 값을 여러 변수에서 공유 X
참조형(Reference Type) : 하나의 객체를 참조값을 통해 여러 변수에서 공유 가능.
ex) b=a면 자바는 값을 복사해서 대입함.

공유 참조와 사이드 이펙트
사이드 이펙트 : 주된 작업 외에 추가적인 부수 효과를 일으키는 것(의도하지 않은 영향)
                    : 같은 객체를 공유하지 않음으로써 해결
                        -> 하지만 강제로 객체 공유를 막을 수 있는 방법은 없음.
=> 불변 객체로 만들어서 변경 불가능하게 하여 해결

불변 객체(Immutable object) : 객체 상태가 변하지 않는 객체
                                        : final, 생성자 사용해서 불변 클래스 만들기

가변 객체 : 상태가 변할 수 있음
불변 객체 : 상태가 변할 수 없음.

불변 객체에서 값 변경하고 싶을 때
-> 결과를 바탕으로 새로운 객체 생성하여 반환
-> 기존 값과 신규값을 모두 확인 가능
